# 真实简历来源筛选表 v1

## 核心假设

- 目标不是找“简历写法文章”，而是找 `可作为测试集原始输入` 的公开简历来源。
- 第一优先级是 `真实性`，第二优先级是 `原始信息密度`，第三优先级才是 `是否完整`。
- 当前公开网络里，`正式岗位 AI PM 的完整原始简历` 很少；更现实的高质量来源是：
  - `求职/求内推帖中的自述型简历文本`
  - `个人站 / GitHub Pages / Notion 在线简历`
  - `人才库 / 公开候选人池`

## 筛选标准

- `真实性`
  - 高：候选人本人公开发布，含明确身份、经历、时间线、求职意向
  - 中：公开人才库/聚合页，有候选人画像，但可能被二次整理
  - 低：教学文章、简历优化文章、机构包装案例
- `原始度`
  - 高：接近用户直接粘贴的正文，有冗余、自述、项目细节
  - 中：页面有一定结构化整理，但仍保留大量候选人原话
  - 低：只剩摘要、评论或方法论
- `岗位适配`
  - 优先 `正式岗位/社招/全职候选人`
  - 排除 `实习生`、`校招模板文`、`简历写作教程`

## 第一批建议保留

- `R01 脉脉求内推帖`
  - 价值：中文、正式岗位、信息密度高、接近真实粘贴文本
  - 用法：优先作为 `国内社招 AI PM` 原始简历样本

- `R02 Web3 人才库 Notion`
  - 价值：候选人画像多，方便挑选不同能力结构
  - 风险：有聚合页整理痕迹，不能直接当“绝对原始简历”
  - 用法：作为 `半原始来源`，适合补多 Agent / Workflow / AI Skill 方向

- `R03 GitHub Pages 个人站`
  - 价值：自维护、经历完整、成果导向强
  - 风险：结构比真实粘贴简历更干净
  - 用法：适合补 `英文 / 海外 / 资深` AI PM 样本

- `R04 GitHub Pages PDF 简历`
  - 价值：最接近真正 resume 文件
  - 风险：目前只确认有 PDF，需要进一步下载核实正文密度
  - 用法：优先进入下一轮抓取

## 明确排除

- 所有 `实习生` 为主的简历来源
- 所有 `简历怎么写`、`简历改法`、`培训/转岗课程` 文章
- 只有提问、没有正文经历的“求建议”帖子
- 明显像机构包装的人才案例页

## 下一步执行顺序

1. 先抓 `R01 / R02 / R03 / R04`
2. 对每个来源做 `原文落盘 + 脱敏`
3. 判断是否满足：
   - 正式岗位候选人
   - 有完整经历或项目细节
   - 不是纯摘要
4. 满足后再进入 `data/test_cases_v1/resume/`

## 备注

- 当前中文互联网里，`公开、正式岗位、完整原始 AI PM 简历` 本来就稀缺。
- 因此下一轮最合理的策略不是继续盲搜，而是：
  - 先把 `R01-R04` 抓下来
  - 看正文密度是否真的够用
  - 不够再补新的来源
